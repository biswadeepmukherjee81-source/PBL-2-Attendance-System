# Blockchain-Based Attendance Management System with Performance Analytics

## 📌 Project Overview
This project implements a blockchain-based attendance management system with performance analytics. It ensures secure, transparent, and tamper-proof recording of attendance data.

The system extends traditional attendance methods by introducing structured dataset handling and analytical computation for performance tracking. Attendance percentage is calculated automatically based on recorded data.

The smart contract is developed using Solidity and tested using Remix IDE. A frontend interface is created and hosted using GitHub Pages to simulate user interaction.

---

## 🚀 Features
- Secure attendance storage using blockchain logic  
- Subject-wise attendance recording  
- Automatic attendance percentage calculation  
- Performance analytics for students  
- Interactive and professional frontend interface  

---

## 🛠️ Technologies Used
- Solidity (Smart Contract)  
- Remix IDE (Development & Testing)  
- HTML, CSS, JavaScript (Frontend)  
- GitHub Pages (Hosting)  

---

## ⚙️ How It Works
1. User enters roll number, name, subject, and attendance status  
2. Attendance is recorded via frontend simulation  
3. Smart contract logic (implemented in Solidity) handles data processing  
4. Attendance percentage is calculated and displayed

## 🔐 Smart Contract
The smart contract is written in Solidity and stored in the file `Attendance.sol`. It contains the contract `AttendanceSystem`, which is responsible for storing attendance records and calculating attendance percentage.

The contract uses structures and arrays to manage student attendance data securely. It ensures transparency and prevents data manipulation, demonstrating the use of blockchain for reliable record keeping.

The smart contract was developed and tested using Remix IDE.
